# 任务后经验维护规则（post-task-learning-review）

完成非平凡任务后，主动使用 $post-task-learning-review 判断并维护本次经验，包括在项目文档、memory、已有 skill 或新 skill 中新增、更新、合并或删除。

对满足维护条件、处于当前任务范围且允许写入的经验，默认直接执行，不再仅提出建议，也不为本地项目文档、允许的 memory 更新或 skill 文件额外请求用户确认。先维护现有内容，再新增，避免重复和过期知识。

项目内经验优先写入项目文档，Codex memory 只作为跨会话补充；写项目文档时遵守渐进式披露。

仍须遵守更高优先级的系统、开发者和用户指令、文件权限、敏感数据规则及外部发布边界。若直接写入受限，使用当前策略允许的更新说明机制；只有确实缺少权限或关键决策时才报告阻塞。
