# 信息可视化呈现规则（show-dont-tell）

## 核心原则

- 能用表格的不用段落。对比、列举、参数、配置 → 表格。
- 能用代码块的不描述代码。命令、配置、SQL、代码片段 → 带语言标注的代码块。
- 能用编号列表的不写连续段落。分步骤操作 → 编号列表，关键命令内嵌代码块。
- 能用树形结构的不写段落。目录结构、层级关系 → 树形图。
- 流程、架构、调用链、状态机 → 用 Mermaid 代码块（`\`\`\`mermaid`），不要用纯文本箭头缩进伪图。
- 正反对比用 ❌/✅ 并排展示，不要埋在段落里。

## 场景对照

| 回复中出现 | 应该用 |
|-----------|--------|
| 对比两个以上选项 | 表格（列=选项，行=维度） |
| 多个属性/参数/字段 | 表格 |
| 分步骤操作 | 编号列表 + 代码块 |
| 目录/层级 | 树形结构 |
| 流程/架构/调用链/状态机 | Mermaid 代码块 |
| 代码/命令/配置 | 代码块（标注语言） |
| 好的写法 vs 坏的写法 | ❌/✅ 并排代码块 |

## 不需要可视化的情况

- 纯观点论述、因果推理 → 用自然段落
- 一两句话能说清的事 → 直接说
- 用户明确要求纯文字
