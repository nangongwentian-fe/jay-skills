# 质量检查清单

## 图片生成前

| 检查项 | 通过条件 |
| --- | --- |
| 页面角色明确 | 已区分封面、章节、内容、图表、对比、结束页。 |
| 图片资产清单存在 | 每张图都有 `id`、`kind`、比例、用途和安全区。 |
| 提示词避开页面文字 | 提示词包含 `no text`、`no logo`、`no watermark`。 |
| 比例匹配摆放方式 | 全幅图用 `16:9`；侧边图常用 `4:3` 或 `16:9`。 |

## 图片生成后

| 检查项 | 通过条件 |
| --- | --- |
| 文件路径稳定 | 图片已经复制到项目内，不只留在生成器临时目录。 |
| 脚本检查通过 | `check_image_assets.py` 退出状态为 `0`。 |
| 人工视觉检查通过 | 没有假文字、商标、水印、畸形主体或弱焦点。 |
| 文字区域可用 | 声明的安全区有足够低细节空间。 |

## PPT 构建检查

| 检查项 | 通过条件 |
| --- | --- |
| 可编辑性 | 标题、正文、表格、图表、结构图都是 PPT 原生元素。 |
| 版式 | 除计划好的全幅背景页外，文字不压住图片框。 |
| 对比度 | 所有图片背景上的文字都可读。 |
| 一致性 | 全套图片风格一致。 |
| 文件体积 | 大图不会在无明显收益时撑大 PPTX。 |

## 最终验证

渲染每页预览，并用缩略总览图检查。重点修这些问题：

| 问题 | 优先改法 |
| --- | --- |
| 主体被裁坏 | 调整裁切焦点，或重新生成构图更明确的图片。 |
| 背景太乱 | 加遮罩或换图片。 |
| 内容太密 | 拆成更多页，不靠缩小字号硬塞。 |
| 页面像截图 | 把文字、图表、结构图重建为 PPT 原生元素。 |
