# Image Enhanced PPT

## 核心规则

把生成图片当成可复用视觉资产，不要当成完整幻灯片，也不要默认只做背景图。

先拆页面，再生成图片：

| 层级 | 例子 | 做法 |
| --- | --- | --- |
| 页面级资产 | 封面主视觉、章节背景 | 适合大面积铺底，需要文字安全区。 |
| 场景级资产 | 产品图、办公场景、行业设备 | 放进固定图片区，辅助说明主题。 |
| 组件级资产 | 流程线、节点、标记、图标、徽章、光效 | 独立生成、独立引用、可复用和组合。 |

这些内容必须保留为 PPT 原生可编辑元素：

| 内容 | 做法 |
| --- | --- |
| 标题和正文 | 用 PPT 原生文本 |
| 数据图表和表格 | 用 PPT 原生图表或表格 |
| 流程图和架构图 | 用 PPT 原生形状或矢量元素 |
| 脚注和来源 | 用 PPT 原生文本 |
| 背景氛围、场景、纹理、产品渲染、小部件视觉皮肤 | 用图片资产 |

## 工作流

1. 先确定故事线、受众、页数和页面类型，再生成图片。
2. 对每页先做组件拆分：哪些是背景，哪些是场景图，哪些是线、点、标记、图标等小部件。
3. 写图片资产计划，包含 `id`、`kind`、`aspectRatio`、`usage`、`safeZone` 和图片提示词。
4. 把生成或收集到的图片放进稳定项目目录，例如 `assets/imagegen/`。
5. 图片里不要出现可读文字、商标、水印、假界面文案、内嵌图表标签。
6. 生成 PPT 前先检查图片资产：

```bash
python3 /path/to/image-enhanced-ppt/scripts/check_image_assets.py \
  --manifest path/to/deck-or-assets.json \
  --report path/to/qa/image-assets.json
```

7. 用 PPT 原生元素控制文字、图表、结构和对齐，用图片小部件提供质感。
8. 渲染预览图，检查整套缩略图；发现问题时改源文件和版式，不改截图像素。

## 图片资产类型

| 资产类型 | 适合用途 | 常见比例 | 备注 |
| --- | --- | --- | --- |
| `hero-image` | 封面、开场页 | `16:9` | 预留干净的文字安全区。 |
| `section-background` | 章节页 | `16:9` | 文字后方需要低细节区域。 |
| `scene-image` | 内容页场景图 | `4:3`、`16:9` | 放在固定图片框内，不压住正文。 |
| `texture` | 轻量背景纹理 | `16:9` | 加遮罩保证文字可读。 |
| `cutout` | 物体或产品图 | `1:1`、透明背景 | 只在版式需要时使用。 |
| `flow-line` | 流程线、时间线、连接线 | `8:1`、`12:1` | 只表达线条质感，不承载方向和文字。 |
| `flow-node` | 流程节点、里程碑点 | `1:1` | 和原生文本框组合，不把文字画进节点。 |
| `marker` | 编号标记、状态点、强调符号 | `1:1` | 可重复使用，保持统一光影和材质。 |
| `icon-asset` | 行业图标、功能图标 | `1:1` | 适合风格化位图图标；普通图标优先用矢量。 |

写或调整资产清单时读取 `references/asset-contract.md`。

## 版式规则

| 页面模式 | 规则 |
| --- | --- |
| 全幅封面 | 图片铺满页面；加深色或浅色遮罩；标题放进声明过的安全区。 |
| 章节页 | 一张强视觉图片、一个短标题、一个可选副标题。 |
| 左文右图 | 固定文字区和图片区；正文不能压住图片框。 |
| 图表页 | 图表和表格用原生元素；图片只做语境补充。 |
| 对比页 | 卡片、分栏、表格用原生元素；图片只做背景或辅助视觉。 |
| 流程图 | 结构由 PPT 原生线框控制；线条质感、节点外观、状态标记可拆成独立图片资产。 |

## 组件拆分规则

| 场景 | 推荐拆分 |
| --- | --- |
| 流程图 | `flow-line` + `flow-node` + 原生文本 |
| 时间线 | `flow-line` + `marker` + 原生日期/说明 |
| 架构图 | 原生容器/连线 + `icon-asset` + `marker` |
| 指标卡 | 原生数字/标签 + `texture` 或 `marker` |
| 产品能力页 | `scene-image` + `icon-asset` + 原生标题/要点 |

组件级图片只负责外观，不负责结构语义。线条的起点、终点、方向、对齐关系应由 PPT 原生形状或布局数据控制。

## 图片提示词规则

| 应该写 | 避免写 |
| --- | --- |
| 明确高级、行业相关的视觉风格。 | 让模型生成完整幻灯片页面。 |
| 明确比例和空白安全区。 | 任何可读文字、标签、商标、水印。 |
| 描述光线、材质、场景、主体。 | 只写抽象渐变，没有实际画面。 |
| 按用途和组件分别生成资产。 | 一张背景图承担整页所有视觉表达。 |

如果有图片生成工具，只让它生成位图资产。PPT 编译器或演示文稿工具继续负责文字、版式、遮罩、图表和导出。

## 验证

使用内置脚本检查文件是否存在、尺寸、比例、非空图像和亮度风险。

这些项目仍需人工看预览确认：

| 检查项 | 原因 |
| --- | --- |
| 多余文字或假界面文案 | 脚本不做文字识别。 |
| 商标、水印、品牌误用 | 属于视觉和合规检查。 |
| 遮罩后的文字对比度 | 取决于实际页面组合。 |
| 裁切质量 | 取决于摆放位置和视觉焦点。 |

交付前读取 `references/quality-checklist.md`。

## 常见问题

| 问题 | 改法 |
| --- | --- |
| 页面像截图 | 把文字、图表、结构图改回 PPT 原生元素。 |
| 只有背景图，没有小部件 | 先按组件拆分，补充流程线、节点、图标、标记等资产。 |
| 图片上文字不可读 | 加遮罩、换安全区，或重新生成留白更干净的图片。 |
| 图片抢主题 | 换更克制的资产、裁切更紧，或改成分栏版式。 |
| 多页风格漂移 | 先定义资产类型和提示词风格，再批量复用。 |
| 文件很难编辑 | 用背景图或场景图替代整页烘焙图片。 |
