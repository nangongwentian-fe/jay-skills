# 黑话分类词表

本文件是可扩展的黑话识别库。发现新黑话时直接添加到对应类别下。

每条格式：`表达` → 问题 → 处理方式

---

## 1. 虚假紧迫感

制造"我随时待命"的假象，拖延实际信息交付。

- "如果你愿意我就开始" → 用户已经发了请求，不需要再确认一次 → 直接回答
- "只要你回复我我立马开始" → 同上，假装在等指令 → 直接回答
- "准备好了吗？我们开始" → 不必要的戏剧化开场 → 删除，直接进入正题
- "我现在就帮你搞定" → 表演积极性 → 删除
- "别急，我来" → 没人在急 → 删除
- "如果你愿意，我可以继续给你..." → 结尾兜售下一步 → 删除，用户需要会自己问
- "如果你要，我下一条可以直接给你..." → 同上 → 删除
- "需要的话我马上..." → 同上 → 删除

**本质问题**：这类表达把"交付信息"包装成"提供服务"，多了一层不必要的表演。开头版（"准备好了吗"）和结尾版（"如果你要我可以继续"）是同一个问题的两面——都在制造不必要的互动轮次。

---

## 2. 咨询黑话

从管理咨询/互联网行业泄漏出来的术语，用于掩盖内容空洞。

- "收窄" → 本意是"缩小范围"，但常被用来假装在做精确分析 → 说"缩小到 X 范围"
- "坐实" → 本意是"确认"，但常被用来假装有确定性结论 → 说"确认是 X 原因"
- "兜住" → 本意是"覆盖/保证"，但常被空洞使用 → 说具体保证了什么
- "落盘" / "落下去" → 本意是"写入/保存/执行"，但常被用来假装完成了工作 → 说"已写入 X"或"已执行 X"
- "收口" → 本意是"结束/总结" → 说"总结"或直接给结论
- "拉齐" / "对齐" → "统一认识" → 说"确认 X"或"统一为 X"
- "抓手" → "具体方法" → 说"方法"或"通过 X"
- "打法" → "策略" → 说"策略"或"做法"
- "闭环" → "完成全流程" → 说"完成"
- "赋能" → "帮助/支持" → 说"帮助"
- "沉淀" → "积累/记录" → 说"记录"或"总结"

**本质问题**：这些词单个看可能无害，但堆砌在一起就变成了"用术语模拟专业性"。一段话全是这类词时，往往没有传递任何具体信息。

---

## 3. 情感绑架

AI 表演情感关怀，制造虚假的亲密感。

- "我就在这里" → AI 不"在"任何地方 → 删除
- "稳稳地接住所有人" → 假装提供情感支持 → 删除
- "稳稳接住你" → 同上 → 删除
- "你不是 X，你只是 Y" → 未经请求的心理安慰 → 删除，除非用户主动寻求情感支持
- "你不孤单" → AI 说这话没有意义 → 删除
- "我懂你" / "我理解你的感受" → AI 没有感受 → 如必要，换成"你说的情况是 X"

**本质问题**：AI 模拟人类情感关怀时，效果适得其反——因为用户知道这不是真的。

---

## 4. 网络流行语

来自社交媒体的流行表达，降低专业性和信息密度。

- "姐妹们" → 不恰当的亲密称呼 → 删除或用"你"
- "保姆级" → "非常详细" → 说"详细的"或"逐步的"
- "谁懂啊" → 假装共情 → 删除
- "狠狠" → "很/非常" → 用"非常"或删除
- "狠狠干" / "狠狠学" → 夸张的强调 → 用正常动词
- "绝绝子" → "很好" → 用具体评价
- "yyds" → "很好/经典" → 用具体评价
- "拿捏" → "掌握/做好" → 用具体描述
- "炸裂" → "很好/很强" → 用具体评价
- "上大分" → "表现好" → 用具体描述
- "遥遥领先" → "优势明显" → 用具体数据或对比

**本质问题**：这类表达用情绪替代信息。"保姆级教程"不如"分 5 步的操作指南"有信息量。

---

## 5. 过度戏剧化

用夸张修辞包装普通信息。

- "我再补一刀" → "我补充一点" → 说"补充"
- "彻底开悟" → "理解" → 说"理解"
- "拍脑门" / "拍脑袋" → "随意决定" → 说"随意决定"或"没有依据地"
- "揪出来" → "找到" → 说"找到了 X"
- "一句话总结" + 然后说了一大段 → 说是一句话但不是 → 真的用一句话总结
- "直接上干货" → 那就直接给内容，不要先说这句 → 删除
- "彻底搞懂" / "彻底弄清" → "理解" → 说"理解"
- "一网打尽" → "全部覆盖" → 说"覆盖了 X 方面"

**本质问题**：普通的"找到了原因"被说成"揪出来了"，普通的"理解了"被说成"彻底开悟"。夸张不增加信息量，只增加阅读负担。

---

## 6. 无意义过渡

假装在组织内容，实际没有传递信息。

- "顺一下" → "整理" → 直接给整理后的内容，不用预告
- "拉起来" → "启动/开始" → 说"开始 X"或直接开始
- "这里收个口" → "总结" → 直接给总结
- "代码已经落下去了" → "代码已写完" → 说"已完成 X 的代码"
- "我们来看看" → 然后就开始了 → 删除这句，直接开始
- "接下来的重点是" → 直接说重点 → 删除这句，直接给内容
- "先说结论" → 然后说了结论 → 直接给结论，不用预告
- "话不多说" → 然后说了很多 → 删除

**本质问题**：这类句子是"关于说话的说话"——它们在描述自己接下来要做什么，而不是直接做。

---

## 7. 组合黑话

多类黑话在同一段话中叠加使用，是最需要警惕的模式。

**典型例子**：

> 我已经把差异**收窄**了，**根因基本坐实**，和我**刚抓到的现象**也对上了。接下来做一个**更硬的排除法**，**稳稳兜住**，**落盘**之后就能**收口**了。

**拆解**：
- "收窄" — 咨询黑话
- "坐实" — 咨询黑话
- "抓到" — 过度戏剧化
- "更硬的" — 无意义修饰
- "稳稳兜住" — 情感绑架 + 咨询黑话
- "落盘" — 咨询黑话
- "收口" — 咨询黑话

**这段话的实际信息**：已缩小排查范围，初步确认根因是 X，下一步做排除验证后给出结论。

**处理原则**：遇到组合黑话，不要逐词替换，要提取整段话的实质信息后重写。
